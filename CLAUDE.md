## Commands

```bash

pnpm build        # Generate tw-version, format, then Vite production build
pnpm format       # Prettier format (CSS, PCSS, JS, JSX)
pnpm cc           # Check formatting with Prettier (no write)
pnpm spell        # CSpell spell check
pnpm release      # Bump patch version, build, and deploy to Vercel
```

## Architecture

Single-page React app displaying auto dealer data fetched from a GraphQL API.

**State flow:**
- `src/utils/store.js` — zustand + persist store; single source of truth persisted to `localStorage` under `autos-r3t::app-data`
- `actions` (exported from store) — call `useStore.getState()`/`setState()` directly; no props or hooks needed
- Components subscribe with `useStore(s => s.field)` selectors

**Data fetching:**
- Two-step GraphQL sequence via `fetchql` against `https://gt-forza.vercel.app/graphql`
  1. Fetch a UUID (`Uuid` query)
  2. Fetch dealer solution using that UUID (`Solution` query)
- `initApp()` — on mount, restores from persisted store or triggers a fresh fetch
- `refresh()` — clears state and re-fetches a new solution

**Key components:**
- `ToolBar.jsx` — dealer picker buttons + refresh trigger
- `DealerView.jsx` — renders `DealerCard` + `VehicleList` for the selected dealer
- `VehicleCard.jsx` — color-coded by vehicle type (exotic/racing/modern/classic)
- `Eswat2Io.jsx` — uses `<proto-ikon-loader>` web component (loaded via CDN in `index.html`)

**Key utilities:**
- `src/utils/store.js` — zustand store + all actions
- `src/utils/tw.js` — class name joiner utility
- `src/utils/tw-version.js` — auto-generated by the `.generate` script; do not edit manually

**Styling:**
- Tailwind CSS 4 with PostCSS; input is `src/styles/input.pcss`, compiled output is `src/styles/app.css`
- Custom color plugin `proto-tailwindcss-clrs` maps named palettes to Tailwind utilities
- Tailwind preflight is disabled in `tailwind.config.js`

**Deployment:** Vercel — `pnpm release` bumps patch version, builds, and deploys.
